Wszystkie grupy Początek :)

  • git config --global user.name ""

    Ustawia nazwę użytkownika, a poprzez zastosowanie funkcji "--global" wystarczy, że taka konfiguracja zostanie dokonana jednorazowo i system automatycznie będzie korzystał z tej nazwy użytkownika.

  • gitk -all

    Wyświetla zmiany w repozytorium lub wybranym zestawie zatwierdzeń. Funkcja "-all" pokazuje wszystkie informacje tj. gałęzie, tagi itp.)

  • git branch "branch" "commit hash"

    Odnajduje w podanej gałęzi commita o podanym kluczu hash.

  • git config --global user.email ""

    Ustawia adres email jakim będzię się posługiwał użytkownik, a poprzez dodanie funkcji "--global" wystarczy, że taka konfiguracja zostanie dokonana jednorazowo i system automatycznie będzie korzystał z tego maila.

  • git checkout -b "branch"

    Tworzy nowe odgałęzienie i automatycznie do niego przechodzi.

  • git branch -d "branch"

    Usuwa gałąź tylko w momencie gdy jest ona aktualnie zmerdżowana.

  • git config --global color.ui true

    Włącza kolorowanie poszczególnych typów obiektów, gdy korzystamy z gita z pozycji terminala.

  • git branch -v

    W trybie listy pokazuje klucz hash oraz commit obiektu wraz z relacją (jeśli taka istnieje) do gałęzi źródłowej, na której się one znajdują. Jeśli zostanie podana dwukrotnie wypisuję również gałęź rodzicielską.

  • git log "branch" --not "branch"

    Dla danej gałęzi odwraca znaczenie przedrostka ^ (lub jego braku) dla wszystkich kolejnych specyfikatorów poprawek, aż do następnego.

  • git commit -a -m ""

    Funkcja pozwala nam na zapisanie zmian w repozytorium, funkcja "-m" pozwala na dodanie opisu do zaistniałych zmian, a "-a" pozwala by każdy zmieniony plik, który jest już śledzony, automatycznie trafił do poczekalni, dzięki czemu pominiemy już część git add.

  • git checkout "branch"

    Pozwala nam się poruszać pomiędzy gałęziami lub przywracać działjące pliki drzewa.

  • git blame -C "filename"

    Pokazuje, wersje oraz autora ostatniej modyfikacji każdego wiersza pliku, a dodanie funkcji "-c" powoduje, że używane jest to samo wyjście co po zastosowaniu "git-annotate"

  • git log --graph --all

    Show commit log. "--all"- pretend as if all the refs in refs/, along with HEAD, are listed on the command line as . "--graph" draw a text-based graphical representation of the commit history on the left hand side of the output.

  • git add .

    Powoduje dodanie aktualnego katalogu roboczego w sposób rekurencyjny do następnego commita. Do commita dodane zostaną pliki znajdujące się w CWD oraz wszystkie pliki z kolejnych podkatalogów w głąb. Można użyć przed każdym commitem jeżeli mamy właściwie skonfigurowany plik .gitignore.

  • git diff --staged

    Pozwala zobaczyć zawartość poczekalni, która trafi do repozytorium z najbliższym zatwierdzeniem.
